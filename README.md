# LEA-Governance-Project
LEA Governance & Pendency Tracking System

## 🧠 Overview
The LEA Governance & Pendency Tracking System is a simulated banking governance project designed to replicate real-world investigation request handling, SLA monitoring, and operational reporting workflows.

It demonstrates how Law Enforcement Agency (LEA) requests are tracked, assigned, monitored, and escalated within a structured governance framework.


## 🎯 Objective
To build a simple governance tracking system that showcases:
- LEA request lifecycle management
- Case pendency tracking
- SLA breach monitoring
- Investigation status visibility
- Operational reporting dashboards


## ⚙️ Tools Used
- Baserow (No-code database & dashboard)
- Excel (Dataset creation)
- Word Docs (SOP documentation)


## 📊 Key Features
- Centralized LEA request tracking system
- Case status management (Open, Pending, Escalated, Closed)
- SLA breach identification
- Investigator assignment tracking
- Dashboard visualization using charts
- SOP-based governance workflow definition


## 📊 Dataset Fields
- Case ID
- Request Type (CCTV, Bank Statement, CIAR, Techno Legal)
- Priority (Critical, High, Medium, Low)
- Status (Open, Pending, Escalated, Closed)
- Assigned Officer
- Days Pending
- SLA Breached (Yes/No)
- Remarks


## 📈 Dashboard Insights
- Request Type Distribution (Pie Chart)
- Investigation Status Overview (Bar Chart)


## 🏦 Business Relevance
This project simulates real-world banking operations where LEA requests must be handled with strict SLA timelines, accountability, escalation control, and audit-ready documentation.
